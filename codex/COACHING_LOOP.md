# Coaching Loop
**Purpose**: A reusable interaction pattern for high-bar learning.

## Core Flow

### Step 1: Orient

Pick the skill:
- behavioral answer
- technical case
- objection handling
- demo explanation
- onboarding question
- executive briefing answer

### Step 2: Narrow the input

Read one small reference.
Do not preload the whole corpus.

### Step 3: Retrieve

Answer from memory.
No scripting.
No reading while answering.

### Step 4: Judge

Return:
- score
- pass / refine / reset
- what built trust
- what weakened trust
- minimum change needed

### Step 5: Repair

Redo the same answer with one deliberate improvement.

### Step 6: Advance

Move on only after the answer clears the bar and survives at least one variant or follow-up.

## Outcomes

### PASS

The answer is:
- correct enough
- concise enough
- structured enough
- trustworthy enough

### REFINE

The answer is directionally right but still weakened by:
- vagueness
- excess length
- poor framing
- weak tradeoffs
- weak close

### RESET

The answer is unusable because it has:
- factual confusion
- no structure
- overclaiming
- evasiveness
- rambling

## Why It Works

It forces:
- active recall instead of passive rereading
- refinement against a visible bar
- progress by demonstrated performance

## Common Failure Modes

- too much reference consumption before first answer
- moving on after a mediocre answer
- polishing language instead of fixing logic
- optimizing for intelligence-sounding instead of trust-building

## Best Use Cases

- interview prep
- onboarding
- technical certification
- product enablement
- objection handling
- demo readiness
- executive briefing prep
